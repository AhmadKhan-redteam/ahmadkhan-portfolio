Projects Lab

Personal projects approached as enterprise case studies — problem, solution, and lessons learned.

Enterprise Monitoring Dashboard
PythonOpenShiftLinux
A monitoring dashboard visualizing CPU usage, memory consumption, deployment health, and application status for enterprise environments.
EXPAND ↓

Problem

Production health signals were scattered across multiple tools, making it slow to spot degraded services.

Solution

Built a single-pane dashboard pulling resource and deployment metrics into one view, surfacing anomalies early.

Architecture

Lightweight Python collectors feed metrics into a web dashboard layer running alongside OpenShift workloads.

Lessons Learned

Centralizing signals reduces mean-time-to-detect — but only if thresholds are tuned to avoid alert fatigue.

Linux Automation Toolkit
PythonBash
A collection of scripts automating routine Linux administration: disk monitoring, log rotation, backups, user audits, and health reports.
EXPAND ↓

Problem

Routine Linux admin tasks were manual, repetitive, and easy to forget under operational load.

Solution

A toolkit of Bash and Python scripts that run on schedule, covering disk checks, log rotation, backups, and audit reports.

Architecture

Cron-triggered scripts write structured logs and reports for quick review.

Lessons Learned

Small, composable scripts are easier to trust and extend than one large automation framework.

OpenShift Cluster Monitor
OpenShiftPython
Monitors cluster health, deployment status, and pod resource utilization using automated scripts.
EXPAND ↓

Problem

Pod-level resource issues often went unnoticed until they affected application availability.

Solution

Automated polling of cluster and pod status, flagging unhealthy workloads before they escalate.

Lessons Learned

Understanding OpenShift's scheduling and health-check model is essential before automating around it.

Incident Response Toolkit
BashSecurity
Automation scripts designed to simplify incident investigation and evidence collection on Linux systems.
EXPAND ↓

Problem

Manual evidence collection during incidents is slow and prone to inconsistency.

Solution

Scripts that capture system state, logs, and artifacts in a consistent, repeatable format during an active incident.

Lessons Learned

Speed and consistency during the first few minutes of response matter more than depth.

Security Audit Toolkit
LinuxSecurity
A Linux security auditing toolkit performing permission checks, user auditing, package validation, and security reporting.
EXPAND ↓

Problem

Misconfigured permissions and stale user accounts are a common source of risk on Linux servers.

Solution

A scripted audit that checks permissions, user accounts, and installed packages, then produces a security report.

Lessons Learned

A short, readable report drives more remediation action than a long raw log dump.
